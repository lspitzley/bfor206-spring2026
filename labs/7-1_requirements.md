# BFOR 206 Lab 7-1: Requirements Gathering & Documentation

## Task Description

The instructor will discuss a problem that he
wants to solve with software. Your task is to gather requirements for a solution to this problem and document them in a Product Requirements Document (PRD). 

Copy the requirements template from the `requirements_template.md` file in the course
repository and fill it out based on the problem described by the instructor. Be sure to include all relevant information, such as the target audience, functional and non-functional requirements, user journey, technical constraints, and any open questions or risks.

## Submission instructions

Fill in details for each section of the PRD.
Show the instructor your completed PRD.
