# BFOR 206 Lab 10-1: Project Testing

## Task Description

Create tests for the area of the project you are responsible for. This may include unit tests, integration tests, or end‑to‑end tests. You should also define the testing strategy and criteria for success.

Work with the requirements defined for your area. Create issues on GitHub if you find
ambiguous or incomplete requirements. Test cases may require data. Test data files
should be added to the repository in a `test_data` directory.

## Submission instructions

1. Create a new branch for your PR (e.g., `exam-testing`). See Lab 8-2 for instructions
on how to create a branch and open a PR.
2. Add outline scripts for testing.
3. Commit your changes with a descriptive message (e.g., "Add test for exam keys").
4. Push your commit to GitHub.
5. Verify that the commit is visible in the Pull Request.
6. In the PR description, explain the changes you made and how they contribute to the project.
7. Request a review from the instructor `@lspitzley` once you have added tests and are ready for feedback.
