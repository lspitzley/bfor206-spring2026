# BFOR 206 Lab 8-1: Pull Requests

## Task Description

Create a Pull Request (PR) on GitHub to prepare your requirements document for the exam generator project. This PR should include:

1. A clear title that reflects the content of the PR (e.g., "Add initial requirements document for exam generator").
2. A detailed description of the changes made, including:
    - An overview of the requirements document.
    - Any specific sections or features added.

## Submission instructions

1. Create a new branch for your PR (e.g., `interface-requirements`).
   - It should be based on the `develop` branch.
2. Add content to the requirements document for your area of the project.
3. Commit your changes with a descriptive message (e.g., "Add names to contributors section").
4. Push your branch to GitHub.
5. Open a Pull Request from your branch to the `develop` branch.
6. In the PR description, explain the changes you made and how they contribute to the project.
7. Request a review from the instructor `@lspitzley`.
